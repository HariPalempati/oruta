Algorithm Used in My Project


HARS can be abbreviated as “Homomorphic Authenticable Ring Signatures”. This mechanism mainly involves with four algorithms: • KeyGen • SignGen • GenProof • VerifyProof

KeyGen : This algorithm mainly generates two keys a public key and a private key. When A tries to send his data onto cloud, In the first step the whole data will passes through this algorithm i.e; this algorithm will be implemented on the data by which the data will be encrypted by producing two keys, a public key and a private key.

SignGen : In the second step, this algorithm will be implemented on encrypted data, by which the whole data will be divided into blocks and each block will gets signed and now these blocks which has a signature and the encrypted data will be placed into the cloud. The whole data is now encrypted.

GenProof : Here A will generate a key by using this algorithm and this key is send to Auditor who is going to verify whether there is any data loss or not.

VerifyProof : This is the final step, where Auditor will request cloud to send the data and then the cloud will respond to auditor by sending a selected blocks which were signed and encrypted. (Note: selection of blocks is random). Now, Auditor will verify the data send by cloud by using this algorithm, whenever auditor implemented this algorithm on the data, a key will be generated. And now he will verify this key and key which was sent by A. If the Key matches, then the whole data is safe and no data loss has been occurred, If the key doesn’t matches, there will be data loss and Auditor notifies A that he should the whole data again.
